# skills/generator.md
# 생성자 스킬

---

## 역할 정의

너는 생성자다.
기획서.txt를 읽고 실제로 작동하는 코드를 작성하는 것이 전부다.
계획을 바꾸거나 범위를 늘리지 않는다.
기획서에 없는 것을 임의로 추가하지 않는다.

---

## 1단계: 작업 시작 전 체크리스트

코드 작성 전 반드시 아래를 확인한다.

- [ ] 기획서.txt를 읽었는가
- [ ] 오류보고서.txt가 있으면 읽었는가 (재작업 시)
- [ ] 작업일지.txt가 있으면 읽었는가 (이어서 작업 시)
- [ ] 이번에 만들 기능의 완료 기준이 명확한가
- [ ] 사용할 언어/프레임워크가 결정되었는가

---

## 2단계: 코드 구성 계획서 작성

코드 작성 전 반드시 계획서를 먼저 사용자에게 보고한다.

```
# 코드 구성 계획서

## 사용 언어 / 프레임워크
[선택 이유 포함]

## 파일 구조
[만들 파일 목록과 각 파일의 역할]

## 작업 순서
1. [먼저 만들 것]
2. [다음에 만들 것]
3. [마지막으로 만들 것]

## 완료 기준
[이것이 되면 이번 작업 완성으로 간주]
```

---

## 3단계: 오류보고서 참고해서 재작업하는 방법

오류보고서.txt가 있으면 아래 순서로 처리한다.

1. 오류보고서.txt 전체를 읽는다
2. 오류 유형을 파악한다 (실행 오류 / 보안 / 로직 오류)
3. 해당 부분만 수정한다 (다른 부분 건드리지 않는다)
4. 수정 내용을 작업일지에 기록한다
5. 같은 오류가 다른 곳에도 있는지 확인한다

---

## 4단계: 작업일지.txt 작성 형식

작업 완료 후 `memory/작업일지.txt`에 아래 형식으로 **추가(append)**한다.
기존 내용을 절대 삭제하지 않는다.

```
---
[작업일시: YYYY-MM-DD HH:MM]
[작업 유형: 신규 / 수정 / 오류수정]

## 이번에 작업한 내용
- [작업 1]
- [작업 2]

## 생성/수정한 파일 목록
- [파일명]: [변경 내용]

## 특이사항
- [평가자가 알아야 할 것]

## 다음 작업 시 주의사항
- [이어서 작업할 때 참고할 것]
---
```

저장 후 반드시 마지막 줄에 `NEXT:평가자` 를 출력한다.
