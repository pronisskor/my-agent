# skills/index.md
# 스킬 인덱스 — 페르소나 ↔ Skill 매핑 정의

이 파일은 agent.ts가 각 페르소나 함수를 실행할 때
**어떤 skill 파일을 반드시 읽어야 하는지** 정의한 기준 문서다.
새로운 skill을 추가할 때 반드시 이 파일을 업데이트한다.

---

## 매핑 테이블

| 페르소나 | 필수 Skill 파일 | 역할 요약 |
|--------|--------------|---------|
| 계획자 | `planner.md` | 요구 분석 → 계획서 작성 → 사용자 확인 루프 |
| 생성자 | `generator.md` | 기술 계획 수립 → 코드 작성 → 작업일지 기록 |
| 평가자 | `evaluator.md` | 정적 분석 → 실행 테스트 → 오류보고서 작성 |

---

## Skill 파일 설명

### planner.md
- 계획자 페르소나가 호출될 때 **반드시** 주입된다.
- 사용자 요구 분석 방법, 계획서 작성 템플릿, 사용자 확인 루프(7단계) 정의.
- 저장 대상 메모리 파일: `memory/기획서.txt`

### generator.md
- 생성자 페르소나가 호출될 때 **반드시** 주입된다.
- 기획서 기반 기술 계획 수립, 코드 작성 지침, 작업일지 기록 방식 정의.
- 저장 대상 메모리 파일: `memory/작업일지.txt`

### evaluator.md
- 평가자 페르소나가 호출될 때 **반드시** 주입된다.
- 정적 분석 체크리스트, CMD 실행 테스트 절차, 오류보고서 작성 형식 정의.
- 저장 대상 메모리 파일: `memory/오류보고서.txt`

---

## Skill 추가 시 규칙

1. `skills/` 폴더에 `{이름}.md` 파일을 생성한다.
2. 이 `index.md`의 매핑 테이블에 해당 페르소나와 파일명을 추가한다.
3. `src/agent.ts`의 해당 페르소나 함수에 `memory.readSkill('{이름}.md')` 한 줄을 추가한다.

## 트리거 키워드 정의

agent.ts의 detectTrigger() 함수가 감지하는 키워드 목록.

| 키워드 | 의미 | 다음 동작 |
|-------|------|---------|
| `NEXT:생성자` | 계획자 → 생성자 전환 | runGenerator() 실행 |
| `NEXT:평가자` | 생성자 → 평가자 전환 | runEvaluator() 실행 |
| `NEXT:계획자` | 어디서든 계획자로 복귀 | runPlanner() 실행, retryCount 초기화 |
| `DONE` | 모든 작업 완료 | 루프 종료, 사용자에게 완료 보고 |
| `STOP:사용자보고` | 3회 재시도 실패 | 루프 강제 중단, 오류보고서 확인 요청 |
| `FEEDBACK:반영중` | 피드백 처리 중 | 사용자 입력 대기 (루프 일시 중단) |
